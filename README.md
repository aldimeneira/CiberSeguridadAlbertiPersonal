# Actividades vulnerabilidades OWASP Top 10

En todas las actividades hay que demostrar la forma de explotar la vulnerabilidad y después mitigarla, demostrando que la vulnerabilidad se ha solventado.

## Relación de actividades:

### Vulnerabilidades de datos de entrada:
- [Inyección SQL](<Vulnerabilidades datos entrada/SQL Injection/>): Ejecución de consultas SQL modificadas por atacante.
- [XSS](<Vulnerabilidades datos entrada/XSS/>) (Cross Site Scripting): Ejecución de código (normalmente javascript) en el cliente.
- [CSRF](<Vulnerabilidades datos entrada/CSRF/>) (Cross-Site Request Forgery): Uso de formularios sin conocimiento del usuario.
- [RCE](<Vulnerabilidades datos entrada/RCE/>) (Remote Code Execution): Ejecución de código en el sistema.
- [Unsafe Deserialization](<Vulnerabilidades datos entrada/Unsafe Deserialization/>): Carga objetos serializados sin validación.
- [SSRF](<Vulnerabilidades datos entrada/SSRF/>): Genera solicitudes desde el servidor.
- [XXE](<Vulnerabilidades datos entrada/XXE/>): Procesamiento de XML sin restricciones.
- [LFI](<Vulnerabilidades datos entrada/LFI/>): Inclusión de archivos del servidor en las peticiones.
- [RFI](<Vulnerabilidades datos entrada/RFI>): Inclusión de archivos en el servidor.
